크롤링

* 본 프로젝트는 java 크롤링 학습용도이며 데이터 저작권으로 인한 불이익은 책임지지 않습니다 : )
 
학습목표 
- 파일 기본 제어 (복사, 삭제 등)
- 웹 데이터 크롤링 

기능구현목표
- 원하는 웹툰 다운로드

사용 라이브러리
- jsoup 1.13.1  : 크롤링
- jfxt  : app ui 관련 라이브러리 (자바 교재 참고)

작업 현황
- 네이버 웹툰 크롤링 기본적인 기능구현. (하드코딩 다수, 코드정리 x)

이슈 사항
- 웹툰명이 겹칠경우 원치않는 웹툰이 크롤링되는 현상이 있음
- 유료 회차 및 로그인이 필요한 웹툰은 크롤링 되지 않습니다. 

이클립스 import방법
1. 원하는 위치에 압축 푼다
2. 이클립스 Project Explorer 마우스 우클릭 import- > import -> General/projects from folder or archive 선택 후 next
3. 상단 directory 버튼 클릭 후 압축 푼 폴더 지정 후 finish


실행방법 

1. (가, 나) 택 1
가. 이클립스에서 luncher패키지 WebtoonAppLuncher.java 파일 실행
나.  jar파일 생성 후 실행 
     프로젝트 우클릭 -> export -> java -> Runnable JAR file 
     -> launch configuration : WebtoonAppLuncher 선택, 다운로드 경로 및 파일명 선택 
     ->  finish -> 생성 파일 더블클릭 

2. 웹툰명 및 저장폴더 회차선택 입력 후 start (100 회당 약 5분가량 소요)

3. 선택 된 폴더에 가보면 html 파일이 있음




패키지 설명 

common 패키지
-> 파일 다운로드 관련 메서드가 정의 되어있음.
* PagingUtil은 삭제해도 무관(페이징 작업하다 귀찮아서 중지)

luncher 패키지 (*** 책보고 헤매면서 만든거라 소스가 특히나 지저분 해요. 포폴 쓰실분은 이 부분 awt, swing활용하여 직접 구현함을 추천드립니다.***)
-> 디자인 구현 및 이벤트 제어 메서드 
-> 해당 패키지 내 java 파일에는 기능이 정의되어있고 fxml 파일에는 디자인이 정의되어있습니다. 
(웹으로 따지면 js - html 관계와 비슷) 

naver 패키지
-> 구현의 핵심 요소가 정의 됨 (main 메서드를 작성 후 start() 메서드를 호출한다면 luncher패키지가 없어도 구동가능)
